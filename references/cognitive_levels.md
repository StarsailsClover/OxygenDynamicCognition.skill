# 认知等级详解

## 概述

Oxygen Dynamic Cognition 定义了四级认知模式，对应不同的推理深度和计算强度。系统会根据问题难度自动选择合适的初始认知等级，并在推理过程中根据置信度动态调整。

## 四级认知模式

### L1 - 快速响应（Fast Response）

**定位**：系统1思维，直觉式快速回答

**适用场景**：
- 简单事实类问题
- 常识性问题
- 是/否判断
- 简单计算
- 无需思考的直接回答

**认知特征**：
- 直接给出答案，不展开推理过程
- 回答简洁，通常 1-3 句话
- 低计算量，快速响应
- 类比人类的直觉反应

**提示词策略**：
- 强调"直接回答"、"简洁"、"不要多余解释"
- 限制回答长度
- 降低 temperature，保证准确性

**典型问题示例**：
- "1+1等于几？"
- "北京是中国的首都吗？"
- "水的化学式是什么？"
- "一年有多少天？"

**预期 Token 消耗**：50-200 tokens

---

### L2 - 分步推理（Step-by-Step Reasoning）

**定位**：基础系统2思维，有条理的逻辑推导

**适用场景**：
- 需要几步推导的逻辑题
- 常规知识解释
- 简单分析类问题
- 需要清晰结构的回答

**认知特征**：
- 先明确问题核心
- 分步骤进行推导
- 每一步有依据
- 最后给出明确结论
- 逻辑链条完整可追溯

**提示词策略**：
- 要求"分步骤"、"先分析再结论"
- 引导结构化输出
- 中等 temperature，兼顾准确性和灵活性

**典型问题示例**：
- "解释什么是光合作用？"
- "比较猫和狗的区别？"
- "如何制作番茄炒蛋？"
- "为什么天空是蓝色的？"

**预期 Token 消耗**：300-800 tokens

---

### L3 - 反思校验（Reflection & Verification）

**定位**：深度系统2思维，自我批判与修正

**适用场景**：
- 复杂问题分析
- 需要严谨性的推理
- 容易出错的计算或逻辑
- 质量要求较高的回答

**认知特征**：
- 先给出初步答案（第一反应）
- 然后站在批判者角度审视
- 从多个维度检查漏洞
- 发现问题并修正
- 最终输出更完善的答案

**自我批判维度**：
1. **逻辑严密性**：有没有推理漏洞？论证是否充分？
2. **事实准确性**：有没有错误或不准确的信息？
3. **角度全面性**：有没有忽略重要因素或视角？
4. **结论可靠性**：有哪些不确定性？适用边界是什么？

**提示词策略**：
- 强制双角色切换：生成者 → 批判者 → 修正者
- 明确的自我检查清单
- 结构化输出：初步答案 → 自我批判 → 修正后答案
- 较低 temperature，保证严谨性

**典型问题示例**：
- "分析量子计算对密码学的影响"
- "设计一个高效的排序算法"
- "解释为什么经济会有周期性波动"
- "评估一个创业项目的可行性"

**预期 Token 消耗**：800-1500 tokens

---

### L4 - 多路径验证（Multi-Path Validation）

**定位**：最高级认知，双路径独立推导 + 交叉验证

**适用场景**：
- 高难度专业问题
- 开放性研究问题
- 需要极高可靠性的结论
- 前沿领域的探索性分析

**认知特征**：
- 从两个完全不同的角度/方法/思路独立推导
- 每条路径都是完整的推理过程
- 然后对比两条路径的结论
- 分析一致性和分歧点
- 综合得出最可靠的最终结论

**路径设计原则**：
- 两条路径必须是**真正独立**的思考方式
- 不能只是同一思路的不同表述
- 理想的路径组合：
  - 理论推导 vs 实证分析
  - 归纳法 vs 演绎法
  - 技术角度 vs 业务角度
  - 宏观视角 vs 微观视角
  - 正向推理 vs 反向验证

**输出结构**：
1. **路径一**：完整推导 + 结论
2. **路径二**：独立推导 + 结论
3. **对比分析**：一致性、分歧点、可靠性评估
4. **最终结论**：综合后的最佳答案 + 置信度说明

**提示词策略**：
- 强制双路径独立思考
- 明确要求路径的差异性
- 结构化对比分析框架
- 中等偏低 temperature，保证严谨性

**典型问题示例**：
- "证明黎曼猜想"（当然，模型无法真的证明，但会用多路径探索）
- "设计通用人工智能的架构方案"
- "预测未来 10 年 AI 的发展方向"
- "分析气候变化对全球经济的长期影响"

**预期 Token 消耗**：1500-2500 tokens

## 认知等级对比表

| 维度 | L1 快速响应 | L2 分步推理 | L3 反思校验 | L4 多路径验证 |
|-----|------------|------------|------------|--------------|
| 认知系统 | 系统1（快） | 系统2（基础） | 系统2（深度） | 系统2（最高级） |
| 推理深度 | 浅 | 中 | 深 | 极深 |
| 自我反思 | 无 | 弱 | 强 | 极强（交叉验证） |
| 典型轮次 | 1轮 | 1轮 | 1轮（含内部反思） | 1轮（双路径） |
| Token 消耗 | ~100 | ~500 | ~1200 | ~2000 |
| 响应速度 | 极快 | 快 | 中 | 慢 |
| 准确率预期 | 70-80% | 80-85% | 85-90% | 90-95% |
| 适用问题 | 简单事实 | 常规知识 | 复杂分析 | 高难度专业 |

## 动态升级机制

### 升级触发条件
当某一轮思考的置信度低于设定阈值时，系统会自动升级认知等级，进行更深入的思考。

### 升级路径
```
L1 → L2 → L3 → L4
```

### 升级策略
1. **第一轮**：使用初始评估的认知等级
2. **后续轮次**：每轮置信度不足时，自动升级一级
3. **达到 L4 后**：继续优化但不再升级等级，改为深度迭代

### 升级时的认知增强
- 不只是换提示词，而是真正的认知深度提升
- 每轮升级都会参考之前的答案，针对薄弱环节深入
- 避免重复劳动，聚焦改进点

## 置信度阈值参考

| 应用场景 | 推荐阈值 | 说明 |
|---------|---------|------|
| 日常对话 | 70-75 | 追求速度，可接受少量不完美 |
| 知识问答 | 75-80 | 平衡速度和质量 |
| 专业分析 | 80-85 | 追求准确性，愿意多花时间 |
| 关键决策 | 85-90 | 高可靠性要求，深度思考 |
| 学术研究 | 90+ | 极致严谨，多轮验证 |

## 最佳实践建议

### 1. 根据场景选择起始等级
- 聊天机器人：默认 L2，简单问题自动降到 L1
- 客服系统：默认 L2，复杂问题升级到 L3
- 代码助手：默认 L3，确保代码质量
- 研究助手：默认 L3 或 L4，追求深度和准确性

### 2. 调整置信度阈值
- 追求效率：降低阈值（如 70）
- 追求质量：提高阈值（如 85）
- 可根据问题类型动态调整阈值

### 3. 限制最大轮次
- 一般场景：3-4 轮足够
- 简单场景：2 轮即可
- 复杂场景：可设 5 轮，但要注意成本

### 4. 结合具体领域优化
- 不同领域的问题，可以定制专门的认知模式
- 例如：数学推理、代码生成、创意写作，各有最优的认知策略
